# Error Handler Skill

## When to Use
This skill is triggered automatically when an audit issue has recurred ≥3 consecutive cycles
and the agent has failed to fix it on its own. It provides a structured diagnostic and fix
workflow to break out of error loops.

## Trigger Conditions
- `DIRECTIVE.md` exists in workspace (auto-generated by audit escalation)
- OR agent detects same audit failure 3+ times in a row
- OR human writes `HUMAN_DIRECTIVE.md` with `⛔` marker

## Mandatory Workflow — DO NOT SKIP ANY STEP

### Step 1: IDENTIFY the Exact Error
- Read the full error text from DIRECTIVE.md or last audit log
- Identify the **error category**:
  - `UNREGISTERED`: Key mismatch between DATASET_MANIFEST.json and code
  - `ORPHAN`: Script bypasses unified data loader
  - `SHORTCUT`: Planned experiment not actually launched
  - `MISSING_MANIFEST`: DATASET_MANIFEST.json doesn't exist
  - `CLASSIFICATION`: Dataset type wrong in manifest
  - `METRIC_MISMATCH`: Reported metrics don't match logs
  - Other: Describe precisely

### Step 2: READ the Files Involved
- **Before making any changes**, read BOTH sides of the mismatch:
  - For `UNREGISTERED`: Read DATASET_MANIFEST.json keys AND the dataset loader code
  - For `ORPHAN`: Read the orphan script AND unified_lf_dataset.py
  - For `CLASSIFICATION`: Read the manifest entry AND verify against domain knowledge

### Step 3: DIAGNOSE Root Cause
Write a one-line diagnosis answering: **What specific string/value/mismatch causes this?**

Examples:
- "Manifest uses key 'Non-lambertian_dataset_zhenglong' but code uses 'Non-lambertian'"
- "Script foo.py calls np.load() directly instead of importing UnifiedLFDataset"
- "Manifest says HCInew is 'Mixed' but it should be 'Lambertian'"

### Step 4: VALIDATE Before Fixing
Run a diagnostic command to confirm your diagnosis:
```bash
python3 -c "import json; m=json.load(open('workspace/DATASET_MANIFEST.json')); print(list(m['datasets'].keys()))"
```
or:
```bash
grep -n 'Non-lambertian' datasets/unified_lf_dataset.py
```

### Step 5: APPLY the Minimal Fix
- Change the **minimum** to resolve the mismatch
- Prefer changing the manifest/config to match code (less risk of breaking training)
- If code must change, change only the specific string

### Step 6: VERIFY the Fix
Run the SAME diagnostic command from Step 4 to confirm the fix works.
Also run a smoke test:
```bash
python3 -c "from datasets import UnifiedLFDataset; print('OK')"
```

### Step 7: CLEANUP
- Delete any temporary test scripts
- Update MEMORY_LOG.md with the fix details
- Verify DIRECTIVE.md has been consumed (archived)

## Anti-Patterns — DO NOT
- ❌ Guess the fix without reading both files
- ❌ Change multiple things at once (makes debugging harder)
- ❌ Skip the validation step
- ❌ Attempt to launch training before the audit passes
- ❌ Rewrite large sections of code to fix a key mismatch

## Success Criteria
- The specific audit issue no longer appears in the next cycle
- No new audit issues introduced by the fix
- Training pipeline still loads data correctly
